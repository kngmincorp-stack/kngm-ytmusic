Troubleshooting

I can’t sign in to my YouTube account

If the sign-in window doesn’t appear or gets stuck, try these steps:

  1. Close all Google Chrome windows completely.
  2. Relaunch YouTube Music Player and try signing in again.
  3. Make sure Google Chrome is installed and set as a default or accessible browser.
  4. If you’re using a company/school network, your IT policy may block the authentication flow. Try on a personal network.

No sound is playing

Audio playback relies on ffplay (bundled with the app). If you hear no sound:

  1. Check that your system volume is not muted.
  2. Make sure the correct audio output device is selected in your Windows sound settings.
  3. Verify that ffplay/ffmpeg is included in the app directory. If the file is missing, re-download the application.
  4. Try playing a different track to rule out a specific file issue.

Songs aren’t loading or take too long

Playback requires an active internet connection. If songs won’t load:

  1. Check your internet connection — try opening a webpage in your browser.
  2. If your connection is fine, try signing out and signing back in to refresh your authentication tokens.
  3. Some regions may experience slower CDN speeds. If a specific track fails, try another and come back to it later.

Listen Together isn’t connecting

The Listen Together feature requires all participants to be on the same local network:

  1. Ensure both the host and listeners are connected to the same Wi-Fi or LAN network.
  2. Check that your firewall is not blocking the application. You may need to allow it through Windows Firewall.
  3. VPNs can interfere with local network discovery. Try disabling your VPN temporarily.
  4. The host must start the session before others can join.

The app crashes or won’t start

If the application won’t launch:

  1. Make sure you’re running Windows 10 or later.
  2. Try running the EXE as administrator (right-click → “Run as administrator”).
  3. Your antivirus may have quarantined the file. Check your antivirus logs and add an exception if needed.
  4. Re-download the application to ensure the file isn’t corrupted.

Still Need Help?
